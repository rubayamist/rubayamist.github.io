Course Content

Module 1

Introduction to Networking Concepts

A computer network connects devices so they can share data and resources. Networks are classified by scale — LAN (Local Area Network), WAN (Wide Area Network) — and by topology, such as star, bus, or mesh.

Example: A university campus network (LAN) connects to the internet (WAN) through a router at the network boundary.
Module 2

OSI & TCP/IP Models

The OSI model describes networking in seven layers (Physical, Data Link, Network, Transport, Session, Presentation, Application), while the more practical TCP/IP model condenses this into four layers. Every network protocol can be mapped to one of these layers.

Example: HTTP operates at the Application layer, TCP at the Transport layer, and IP at the Network layer.
osi_layers = [
    "7 - Application",
    "6 - Presentation",
    "5 - Session",
    "4 - Transport",
    "3 - Network",
    "2 - Data Link",
    "1 - Physical",
]

for layer in osi_layers:
    print(layer)
Module 3

IP Addressing & Subnetting

Every device on a network needs a unique IP address. Subnetting divides a large network into smaller sub-networks, each defined by a network address, broadcast address, and range of usable host addresses.

Example: Find the network address and usable host range for the IP address 192.168.1.50 with subnet mask 255.255.255.0.
import ipaddress

network = ipaddress.ip_network("192.168.1.50/24", strict=False)

print(f"Network address : {network.network_address}")
print(f"Broadcast address: {network.broadcast_address}")
print(f"Usable hosts     : {network.num_addresses - 2}")
Module 4

Routing & Switching Basics

Routers forward data between different networks based on IP addresses, while switches forward data within a network based on MAC addresses. A routing table tells a router where to send traffic for each destination network.

Example: A simple static route configuration on a router.
! Example Cisco-style static route configuration
Router(config)# ip route 192.168.2.0 255.255.255.0 192.168.1.1
! Sends traffic destined for 192.168.2.0/24 network
! to the next-hop router at 192.168.1.1
Module 5

Network Security Fundamentals

Network security protects data as it travels across a network, using techniques such as encryption, firewalls, and access control. Encryption scrambles data so only the intended recipient can read it.

Example: A simplified substitution cipher demonstrates the basic idea behind encryption (not suitable for real security use).
def caesar_encrypt(text, shift):
    result = ""
    for char in text:
        if char.isalpha():
            base = ord('A') if char.isupper() else ord('a')
            result += chr((ord(char) - base + shift) % 26 + base)
        else:
            result += char
    return result

message = "NETWORK"
encrypted = caesar_encrypt(message, 3)
print(f"Original : {message}")
print(f"Encrypted: {encrypted}")
Module 6

Hands-on Networking Exercises

Socket programming lets you write applications that communicate over a network directly. This exercise builds a minimal client-server pair using Python's built-in 'socket' module.

Example: A basic TCP server that echoes back any message it receives, and a client that sends one.
# server.py
import socket

server = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
server.bind(("localhost", 5000))
server.listen(1)
print("Server listening on port 5000...")

conn, addr = server.accept()
data = conn.recv(1024).decode()
print(f"Received: {data}")
conn.send(data.encode())   # echo back
conn.close()


# client.py
import socket

client = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
client.connect(("localhost", 5000))
client.send("Hello, server!".encode())
response = client.recv(1024).decode()
print(f"Server replied: {response}")
client.close()
